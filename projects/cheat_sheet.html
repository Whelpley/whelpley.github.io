Looping in Ruby vs JavaScript

Looping is an essential part of programming. Ruby and JavaScript both have ways for repeating repeating repeating commands, but they're a little different.

This guide will provide a side-by-side comparison of common loop commands in both languages.

Ruby

Basic Loop

loop {Code_block}
// Alternatively:
loop do
Code_block
end
  • Will execute Code_block repeatedly, unless a break condition is introduced. This is an infinite loop by default.

While

while Condition
Code_block
end
// A one-line version:
Code_block while Condition
// A reverse-execution version:
begin
Code_block
end while Condition
  • In the first two versions above, the Condition is evaluated, and if True, the Code_block executes. The program then loops back to checking the Condition.
  • In the third case, the Code_block is executed, then the Condition is checked before looping back. In this version, the Code_block will always be executed at least once.
  • This can easily become an infinite loop, i.e. if you neglect to base the Condition on something changed in the Code_block.

Until

until Condition
Code_block
end
// A one-line version:
Code_block until Condition
// A reverse-execution version:
begin
Code_block
end until Condition
  • This works like a reverse version of While, in that the Condition is evaluated, and if False, the Code_block will be executed.

Times

X.times {Code_block}
// With an added capability:
X.times {|i| Code_block}
  • In the first version, this will execute the Code_block X number of times.
  • In the second version, the variable i is available for each iteration of the Code_block. The value of i is equal 0 the first iteration, and increments by one each successive iteration.

For

for Element in Array
Code_block
end
  • This loop will evaluate the Code_block on each Element in an Array.
  • The Code_block does not necessarily need to have any effect on the elements of the array.

Enumerable #each

// Array version:
Array.each {|element| Code_block}
// alternatively:
Array.each do |element|
Code_block
end
// Hash version:
Hash.each {|key, value| Code_block}
// alternatively:
Hash.each do |key, value|
Code_block
end
  • This is the most basic of the Enumerable methods, which loop code over each part of a grouping of variables.
  • In the Array version, the Code_block is executed over each element in the array.
  • In the Hash version, the Code_block is executed over each key in the hash. During each iteration, the value for that key is made available to the Code_block.
  • This is just the tip of the iceberg regarding Enumerable methods; for a detailed listing you may refer to the Ruby Docs.

JavaScript

For

for (Statement1; Statement2; Statement3) {
Code_block;
}
// Statement1 is executed before Code_block starts
// Statement2 defines the condition (which must evaluate to true) to keep running Code_block
// Statement3 is executed each time after Code_block has been executed
  • Statement1 is normally used to initiate the variable used in the loop, but is optional.
  • You may initiate multiple values in Statement1, as long as they are separated by commas.
  • Statement2 is usually used to check the condition on the initial variable.
  • Statement2 is also optional, but a break condition must otherwise be introduced to prevent an infinite loop.
  • Statement3 is usually used to increment the initial variable, but may perform other operations as well.
  • Statement3 is likewise optional, especially if the initial variable is changed in the Code_block.

For/In

for (Property in Object) {
Code_block;
}
  • This loops through each Property in an Object, enacting the Code_block on each property.

While

while (Condition) {
Code_block;
}
  • The Condition is evaluated, if True, then Code_block is executed, then loops back to Condition check.
  • To prevent infinite loops, be sure to change the variable used in Condition, or otherwise introduce a break.

Do/While

do {
Code_block;
}
while (Condition)
  • Code_block is executed, then Condition is checked; if it evaluates to True, loops back to Code_block.
  • The difference between this loop and the While loop above is that the Code_block Do/While is evaluated at least once, while in the While, it may never execute.
