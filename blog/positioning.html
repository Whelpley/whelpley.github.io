Positioning in CSS

The difference between Static, Absolute, Fixed, and Relative positioning

October 18, 2015

In this past week's studies for Dev Bootcamp, I received a crash course in CSS. CSS stands for Cascading Style Sheets, and it is the language that is used to create a styling guide for an HTML document on the World Wide Web.

One of the more difficult concepts I grappled with this week was Positioning: the rules that govern where an HTML object appears in the screen. After coming to an understanding of their properties and differences, I have chose to share my knowlege via this post.

To begin with, I will further explain the relation between CSS and HTML. An HTML document is composed of "elements", which are the individual components of a page, as defined by its creator. The HTML document may include a link to a CSS "stylesheet", where elements may be picked out and have a "property" and "value" applied to them.

There is a large range of Properties one may apply to the elements, which can change the font size, borders, colors, and many other factors. In this essay, I will stick to covering the Position property, which governs where the element will appear on a page. You may select one of four values for this property:

  1. Static: This is the default positioning value, which will be applied when no other is specified. This means that each element will appear in the order it appears in the HTML document.
  2. Relative: When selected on its own, this position with work the same as Static. However, you may also apply additional properties (eg: top: 5px, bottom: 2em) to shift the element from where it would have appeared in Static. Of note, selecting this position will increase the element's "z-axis" value, which means it will appear on top of any Static element.
  3. Absolute: when this position is selected, the element will start from the nearest parent element (one whch contains the element in question) with Absolute or Relative positioning, or the start of the document body if no such parent exists. The element may then be shifted with addition top:/bottom:/etc properties, as with Relative.
  4. Fixed: This is the odd one of the bunch - when this property is selected, the element becomes fixed to a specific point in the browser window, so that as the page is scolled, that element remains in the same space. This one must be used carefully to avoid overlapping elements.

I have drawn up an example to demonstrate the differences in the code; you may refer to the style sheet here. The yellow box is Absolute, further down, and the red box is Fixed, which means it will stay in the same place no matter where the browser window is scrolled.
