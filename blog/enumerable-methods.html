Enumerable#cycle

An introduction and explanation

November 1, 2015

This past week at Dev Bootcamp, Phase 0 was our second (out of three) covering Ruby. We dived deeper into Enumerable methods, which are methods that iteratively enact code blocks on data collections (such as arrays, hashes, and ranges).

In this blog post, I will cover one of the methods that I discovered this week: Enumerable#cycle. This is a method that iterates repeatedly over the elements of an enumerable, running infinitely unless a stop condition is acheived.

Implementing this method in an Arrays looks like this:

array_name.cycle(n) {|element| block}

When this method is called, the code in "block" is applied to each "element" in the array "n" times. If "n" is not specified (or set to nil), the method will infinitely iterate over the array, starting from the beginning after the end. The only other way to end the loop is to include a Break clause in the code block for a specific state.

With its potential for infinite loops, this method should be used with caution. In the example below, I will show how this can be used in combination with the methods .next and .rewind to access the months in a year:

Month Cycle

Here, I declare an array with the months in a year, then declare a variable "this_month". Each time Next is called on this_month, it returns the next one in the cycle. When Rewind is called, the cycle returns to the start:
months_of_year = %w{ Jan Feb Mar Apr May Jun Jul Aug Sep Oct Nov Dec}
this_month = months_of_year.cycle
this_month.next
# => "Jan"
this_month.next
# => "Feb"
this_month.rewind
this_month.next
# => "Jan"

Enumerable#cycle works similarly for hashes, with a small adjustment to the syntax:

hash_name.cycle(n) {|key, value| block}

I would like to conclude today's post with a finer note about the Enumerable methods, which took a while to sink in this week: while they all provide means of enacting a block of code on a data structure, when they return when called is generally Nil. In order to make use of the results of the code block, they have to be "caught" by another variable in the code block. The example below should demonstrate:

Returns from Enumerable methods

First, I will declare an array, and then call a cycle method that acts on the array, but does not otherwise catch the results:
ex_array = [2, 4, 6]
ex_array.cycle(2) {|x| x+2}
# => nil
Now, we funnel the results of the code block into something we can see:
ex_array = [2, 4, 6]
ex_array.cycle(2) {|x| p x+2}
# => 4
# => 6
# => 8
# => 4
# => 6
# => 8

The above example points out one other feature of Enumerable#cycle, in that when the code is repeatedly called over a data structure, it returns to the original structure, rather than the results of the first (and subsequent) passes over the structure.

I hope this post helped illuminate this method, and that you might put it to use in your own coding adventures. See you next week!.
