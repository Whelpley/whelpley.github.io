Arrays and Hashes

What's the difference?

October 25, 2015

Welcome back! Today's blog will cover Arrays and Hashes, with respect to the Ruby programming language. I will try to keep the description acessible to those not schooled in programming, though I will also include code examples as a demonstration.

Arrays and Hashes are two means of storing collections of data. Both enable the data to be sorted and otherwise operated on, but they also have a few key differences.

Arrays

To envision an array, think of a row of buckets. Each bucket can contain any other thing, and each one is labeled by its order in the row. When you load up the buckets with the data you are trying to store, each item gets placed in its own bucket, in sequential numerical order (unless otherwise specified), starting with slot "0" and going up. When you want to later access the contents of a bucket, you tell the array-conveyor to bring you a particular bucket via its index number.

I will demonstrate a few Ruby-specific things about Arrays in the code examples below:

Array Examples

Here I am declaring a new array, with 3 slots. Note how each slot has a different type of data (integer, float, string). That's OK!
new_array = [45, -3564.043, "walking dogs"]
Having declared the new_array, I can call it like so below to return the value "walking dogs". (Remember that the array index starts at [0], so the third slot will be [2]:
new_array[2]
# returns the string "walking dogs"
There are many methods you can call to manipulate the contents of an array; one simple example is "reverse", which reverses the order of the array:
new_array.reverse!
new_array[2]
# returns the integer 45

Hashes

A hash, on the other hand, is often analogized to a dictionary. Every hash consists of a collection of pairs of objects, called a "key" and a "value". The key is like the word which the dictionary is sorted by, and the value is like the definition of that word. So, rather than accessing a value by its numerical order amongst other data, you can just remember the name of the key, and you can access that data associated with that key. Most importantly, a hash can only have one of any type of key, but multiple keys may point to the same value.

As before, I will demonstrate a few Ruby-specific things about Hashes in the code examples below:

Hash Snippet

Here is one way you can create an hash, using what is called Literal Construction. Each key and value gets specifically indicated, with the key on the left column and the value to the right of the => symbol:
new_hash = {
"name" => "Michael",
"state" => "CA",
"signature move" => "The Windmill",
}
To access the contents of an hash, you call its key, rather than the index number. (There are ways to do the latter in recent versions of Ruby, however.)
new_hash["state"]
# returns the string "CA"
As with arrays, you can call methods on the hash to manipulate it. Here, we will employ "value?", which returns "true" if the hash contains the specified value.
new_hash.value?("The Windmill")
# returns True

Aside from this basic structural difference, there are some features which can make one or the other more desirable to use:

  • An array is preferable when you have a set of one type of data, while a hash makes more sense for pieces of data that are paired.
  • Arrays are easier to access and sort by insertion order.
  • If you wish to combine data sets, arrays are much easier to put together. With hashes, you run the risk of merging overlapping keys, which may result in data loss.
  • For reasons not yet understood by me, hashes are quicker to access, which can improve the speed of processes using them over arrays.

I recognize that I have only touched the surface of these topics, and one can go much further in depth here. Nevertheless, I hope you learned something, and that you will come back around next week!
